# NSMC 영화 리뷰 감성 분류

네이버 영화 리뷰를 기반으로 리뷰가 긍정(`POSITIVE`)인지 부정(`NEGATIVE`)인지 예측하는 이진 분류 프로젝트.

이 프로젝트는 딥러닝 모델을 사용하지 않고, 한국어 텍스트 전처리와 TF-IDF 기반 모델만으로 높은 점수를 내는 것을 목표로 했다. 최종 모델은 Word/Char TF-IDF에 Naive Bayes 로그 비율을 곱한 뒤 LinearSVC로 분류하는 NBSVM 방식이다.

## 분석 과정

### 1. EDA

- 라벨 분포와 텍스트 길이 분포 확인
- 중복 리뷰와 라벨 충돌 데이터 탐색
- 욕설, 반복 기호, `ㅋㅋ`, `ㅠㅠ`, `!!`, `??` 같은 표현이 감성에 주는 영향 확인

리뷰 길이만으로는 긍/부정을 강하게 나누기 어렵지만, 욕설과 강조 기호는 감성 신호로 쓸 여지가 있었다. 그래서 단순 정제보다는 한국어 리뷰 특유의 표현을 최대한 살리는 방향으로 전처리를 설계했다.

### 2. 데이터 클리닝

| 처리 | 설명 |
|------|------|
| 중복 제거 | 동일한 `text` 중복 제거 |
| HTML/CSV 아티팩트 제거 | `<br>`, 반복 따옴표 등 정리 |
| 무의미 텍스트 제거 | 자모, 기호, 숫자만 있는 리뷰 제거 |
| 라벨 충돌 제거 | 같은 텍스트에 서로 다른 라벨이 붙은 경우 제거 |

클리닝 후 학습 데이터는 **145,930건**이 남았다.

### 3. 텍스트 정규화

한국어 영화 리뷰는 표준어보다 구어체, 오타, 욕설 변형, 신조어가 성능에 더 크게 영향을 줬다. 이 부분을 정규식 기반으로 직접 정리했다.

| 정규화 항목 | 예시 |
|-------------|------|
| 욕설 변형 통일 | `씨발`, `시발`, `ㅅㅂ` → `시발` |
| 신조어 정리 | `꿀잼`, `존잼`, `개잼` → `재미있다` |
| 부정 표현 정리 | `노잼`, `잼없` → `재미없다` |
| 오타 교정 | `별루` → `별로`, `조타` → `좋다` |
| 반복 기호 축약 | `ㅋㅋㅋㅋㅋ` → `ㅋㅋ`, `ㅠㅠㅠㅠ` → `ㅠㅠ` |
| 역접 강조 | `지루하지만 재밌다` → 뒤쪽 감성어 가중 |

특히 `하지만`, `근데`, `는데` 뒤에 나오는 표현이 실제 감성을 뒤집는 경우가 많아, 역접 뒤 감성어를 한 번 더 강조했다.

### 4. 형태소 분석

형태소 분석기는 `kiwipiepy`를 사용했다.

- 영화 리뷰에 자주 나오는 단어를 사용자 사전에 추가
- 조사, 어미처럼 감성 판단에 약한 품사는 제거
- 명사, 동사, 형용사, 부사, 감탄사 중심으로 토큰 유지
- `안 좋다`, `재미 없다` 같은 표현은 부정 의미가 깨지지 않도록 합성 토큰으로 처리

예를 들어 `안 좋다`를 `안`, `좋다`로 분리하면 `좋다`가 긍정 신호로 잘못 작동할 수 있다. 그래서 `안_좋다`, `재미_없다`처럼 묶어 부정 의미를 보존했다.

## 모델링

### 최종 모델: NBSVM

최종 모델은 단일 모델이다. 스태킹이나 딥러닝을 쓰지 않고, TF-IDF 피처를 최대한 잘 만드는 쪽에 집중했다.

```
정규화된 원문
├── Kiwi 토큰화 → Word TF-IDF
└── 원문 유지   → Char TF-IDF
        ↓
Naive Bayes 로그 비율 가중
        ↓
Word + Char 피처 결합
        ↓
LinearSVC
        ↓
CalibratedClassifierCV
        ↓
threshold 적용
```

### 왜 TF-IDF인가?

이 데이터는 짧은 리뷰가 많고, 감성 단어와 표현 변형이 점수에 직접적으로 영향을 준다. 그래서 문맥을 길게 이해하는 모델보다, `재미없다`, `시간낭비`, `강추`, `노답`, `졸작` 같은 표현을 안정적으로 잡는 것이 중요했다.

Word TF-IDF는 형태소 단위의 의미를 잡고, Char TF-IDF는 오타나 변형 표기를 보완한다. 여기에 NB 로그 비율을 곱해 긍정/부정 한쪽에 강하게 치우친 표현의 가중치를 키웠다.

### 주요 하이퍼파라미터

Optuna로 60 trials, 5-fold OOF MCC를 기준으로 탐색했다.

| 항목 | 값 |
|------|----|
| Word n-gram | `(1, 4)` |
| Word min_df | `3` |
| Word max_features | `200,000` |
| Char n-gram | `(2, 5)` |
| Char min_df | `3` |
| Char max_features | `400,000` |
| NB alpha | `0.1243` |
| Char 가중치 | `1.3622` |
| LinearSVC C | `0.2231` |
| LinearSVC loss | `squared_hinge` |
| Threshold | `0.5445` |

## 성공 요인

1. **딥러닝보다 데이터 표현에 집중**
   - 리뷰가 짧고 감성 표현이 직접적인 데이터라 TF-IDF가 잘 맞았던것 같다.

2. **Word + Char TF-IDF 결합**
   - Word TF-IDF는 의미 단위를, Char TF-IDF는 오타와 변형 표현을 잡아 서로의 약점을 보완했다.

3. **한국어 리뷰 특화 전처리**
   - 욕설, 신조어, 오타, 반복 기호를 통일해 같은 의미의 표현이 여러 단어로 흩어지는 문제를 줄였다.

4. **부정 표현 보존**
   - `안_좋다`, `재미_없다`처럼 부정 의미를 묶어 긍정 단어가 잘못 작동하는 경우를 줄였다.

5. **임계값 최적화**
   - 기본 threshold 0.5를 그대로 쓰지 않고 MCC 기준으로 threshold를 조정했다.

## 한계와 개선 여지

- 긴 문맥의 반전 감성은 여전히 어렵다.
  - 예: `초반은 좋았는데 뒤로 갈수록 별로였다`
- 풍자와 반어법은 TF-IDF만으로 안정적으로 잡기 어렵다.
  - 예: `와 진짜 대단한 영화네요...`
- 정규식 기반 전처리는 점수 개선에는 효과적이지만, 규칙이 많아질수록 유지보수가 어려워진다.

딥러닝 모델을 쓰면 더 높은 점수를 노릴 수는 있지만, 이 프로젝트에서는 의도적으로 TF-IDF 기반 접근만 사용했다. 결과적으로 단순한 모델도 전처리와 피처 설계를 잘하면 충분히 경쟁력 있는 성능을 낼 수 있음을 확인했다.
